Source: https://www.kaggle.com/datasets/sarveshchhetri/e-commerce-return-abuse-detection-dataset/data

This dataset is a synthetic dataset that contains **60,000 highly detailed e-commerce return transactions**, meticulously featuring customer demographics, behavioral metrics, and shipping information. The target variable is `abuse_type`, an imbalanced multi-class label categorized into:
* **Legitimate (70.1%)**: Normal, honest returns.
* **Policy Abuser (11.9%)**: Customers who exploit return policies (e.g., returning 90% of what they buy).
* **Fraudulent Return (10.2%)**: Malicious returns (e.g., sending back empty boxes, tracking fraud).
* **Wardrobing (7.7%)**: Buying items for a one-time event and returning them.

With **zero missing values** and a rich set of 35 features, this dataset is an exceptional playground for feature engineering, anomaly detection, and predictive modeling.
